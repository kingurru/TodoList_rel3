Задание по Курсовому проекту:

Необходимо разработать систему планирование ваших дел и задач ("Планировщик задач").

Этапы разработки:
1) -[x] Создать отдельный репозиторий для проекта (для тех, кто еще не сделал этого, те кто выполнял промежуточные задания это делать не надо). 
   Каждая подзадача, начиная с пункта "2" делается в отдельной ветке и мне на проверку создаете Pull Request. 
   Где я буду проверять задачи, оставлять комментарии.


2) -[x] Развернуть webpack как это делали в двух предыдущих лекциях.


3) - [x] Добавить возможность создания задачи, изначально задач нет. Для создания задачи, необходимо добавить форму, где в input
   указывается имя задачи, также добавить select (в нем есть выбор приоритета задачи: Не важные, Важные, Очень важные). Благодаря select
   можно указать приоритет задачи. Эти два поля input и select являются обязательными, без заполнения их задача не должна создаваться. 
   Нельзя также создавать задачи с одинаковыми названиями в рамках всех трех категорий. 
   Также на форме должна быть кнопка с типом submit, для обработки события отправки данных. 


4) - [x] Создаваемые задачи в зависимости от категории их приоритета должны отображаться в трех столбцах. У каждого столбца должен быть
   заголовок соответствующего цвета:

    - Зеленый самый левый столбец - "Не Важные"
    - Желтый средний столбец - "Важные задачи"
    - Красный самый правый столбец - "Самые важные задачи"


5) - [ ] Для тех, кто делал промежуточные задания. Кнопки "Показать" и "Скрыть" убирают. Теперь скрывать и показывать можно будет каждый столбик
   в отдельности. Для этого используйте обработку клика по заголовку столбца как в резюме сделали.


6) - [ ] Каждую задачу можно пометить как сделанную благодаря checkbox слева от текста задачи. Поэтому показывать и скрываться будут только выполненные задачи.


7) - [ ] Справа от текста задачи добавить кнопку удалить, чтобы была возможность удалить задачу. При этому удалить можно ту задачу, которая
   помечена checkbox как сделанная.


8) - [ ] Для хранения списка задач используйте localStorage. В котором будет храниться три массива задач по каждой категории в формате JSON.


9) - [ ] В качестве дополнительного задания рассмотрите возможность редактирования каждой задачи. Изменения ее имени и (или) перемещения в другую категорию.
